6. Structure the troubleshooting steps as numbered items
7. Keep ALL original content — don't remove anything, only add structure

[SYMPTOMS] <what the user sees or reports>

[TROUBLESHOOTING]
1. First step

(keep all original steps, clean up formatting)

[RESOLUTION]
What fixes the issue (summarize from article)

When to escalate: <conditions>
Info to include: <what to put in escalation notes>

[ORIGINAL_ARTICLE]
<paste the complete original text here unchanged>

---

IMPORTANT RULES:
- ALWAYS include Spanish keywords/synonyms (our team supports retail in both languages)
- Keywords should include: error messages, system names, common misspellings, related tools
- Be generous with synonyms — the more ways you describe the problem, the better the search match
- If the article doesn't specify an escalation team, write "Check SNOW assignment groups"
- Keep troubleshooting steps actionable and numbered
- If the article is vague, still extract what you can and note gaps

Zeus Login Issue
